API NutriDoc

L'API NutriDoc permet aux partenaires, prescripteurs et structures d'intégrer nativement les services nutritionnels certifiés RPPS dans leurs propres applications (CRM, logiciels de gestion, applications mobiles).

L'API suit les principes RESTful, retourne des réponses en JSON et utilise des tokens JWT pour l'authentification.

Base URL

https://api.nutridoc.calidoc-sante.fr/v1/

Versioning

L'API utilise le versioning dans l'URL (/v1/). Les versions majeures sont annoncées 6 mois à l'avance.

Authentification

Toutes les requêtes API doivent inclure un token JWT dans l'en-tête Authorization.

Obtenir un token

POST /auth/token 🔒 Public

Corps de la requête

{
  "api_key": "votre_api_key",
  "api_secret": "votre_api_secret"
}

Réponse

{
  "access_token": "eyJhbGciOiJIUzI1NiIs...",
  "token_type": "Bearer",
  "expires_in": 3600
}

Utiliser le token

curl -X GET "https://api.nutridoc.calidoc-sante.fr/v1/plans" \
  -H "Authorization: Bearer votre_token_jwt"

Rate Limits

Les limites sont appliquées par token API :

  • Plan gratuit : 100 requêtes / heure
  • Plan Pro : 1 000 requêtes / heure
  • Plan Enterprise : 10 000 requêtes / heure

En cas de dépassement, l'API retourne un code HTTP 429 Too Many Requests avec l'en-tête Retry-After.

🔔 Webhooks

NutriDoc peut envoyer des notifications en temps réel sur vos endpoints configurés.

Événements disponibles

ÉvénementDescription
plan.completedPlan alimentaire livré
plan.pendingNouvelle commande de plan
patient.createdNouveau patient créé
credits.lowCrédits inférieurs à 20%
visio.scheduledVisio programmée

Payload exemple

{
  "event": "plan.completed",
  "timestamp": "2026-04-13T10:00:00Z",
  "data": {
    "plan_id": "plan_789",
    "patient_id": "pat_456",
    "pdf_url": "https://..."
  }
}

Configuration

POST /webhooks/register
{
  "url": "https://votre-site.com/webhook",
  "events": ["plan.completed", "plan.pending"]
}

📦 SDK & Bibliothèques

Des SDK officiels sont disponibles pour faciliter l'intégration :

  • JavaScript/Node.js : npm install nutridoc-api
  • Python : pip install nutridoc-api
  • PHP : composer require nutridoc/api

Exemple JavaScript

const NutriDoc = require('nutridoc-api');

const client = new NutriDoc({
  apiKey: 'votre_api_key',
  apiSecret: 'votre_api_secret'
});

const plan = await client.plans.create({
  patient: { prenom: 'Marie', email: 'marie@email.fr' },
  objectif: 'perte_poids'
});
